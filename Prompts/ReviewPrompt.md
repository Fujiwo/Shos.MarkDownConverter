# Shos.MarkDownConverter レビュープロンプト for GitHub Copilot

あなたは GitHub Copilot として、このリポジトリ全体のレビューを行ってください。
表面的な感想ではなく、実際の不具合、回帰リスク、要件未達、設計上の不整合、テスト不足を中心に確認してください。

まず既存のリポジトリ構成と主要ファイルを確認し、推測ではなく実装と設定に基づいてレビューしてください。

## 1. レビュー対象

対象はリポジトリ全体ですが、特に次を重点的に確認してください。

1. ASP.NET Core Web アプリ本体
2. Python 版 MarkItDown 呼び出し処理
3. ユーザー向けエラー表示
4. API と UI の契約
5. 単体テスト、結合テスト、E2E テスト
6. README と詳細ドキュメント

## 2. プロジェクト前提

Shos.MarkDownConverter は、ブラウザーから単一ファイルをアップロードし、サーバー側で Python 版 MarkItDown を実行して Markdown に変換する Web アプリケーションです。

この前提を踏まえ、少なくとも次が正しく成立しているかを確認してください。

1. 単一ファイルアップロードが動作すること
2. Python 版 MarkItDown を安全に呼び出していること
3. 変換結果を画面表示、コピー、ダウンロードできること
4. 異常時に適切なエラーが表示されること
5. ドキュメントと実装が矛盾していないこと

## 3. 重点レビュー観点

以下の順で優先して確認してください。

1. 実行時に壊れる可能性がある箇所
2. ユーザー向け挙動として不自然または誤解を招く箇所
3. 例外処理やエラーメッセージの不備
4. セキュリティ上の懸念
5. API と UI の契約不整合
6. テスト不足や検証漏れ
7. 保守性や可読性の低い実装
8. 不要な複雑化や設計上の違和感

特に次は必ず確認してください。

1. Python 実行パスや MarkItDown 呼び出しに関する設定の扱い
2. ProcessStartInfo.ArgumentList を使った安全な引数処理
3. 一時ディレクトリと一時ファイルのクリーンアップ
4. 標準出力、標準エラー、終了コードの扱い
5. 変換失敗時の UI 文言が、実際の失敗原因と一致しているか
6. 通信失敗、入力不備、環境不備、変換失敗が混同されていないか
7. E2E テスト基盤が不安定になりやすい箇所がないか

## 4. エラー処理レビュー方針

ユーザー向けエラー表示は重点的にレビューしてください。

次の観点を必ず確認してください。

1. 何が起きたかが短く明確に書かれているか
2. 原因候補が実際の失敗種類に合っているか
3. 対処方法が具体的で、次に何をすればよいか分かるか
4. 通信失敗とサーバー内部失敗が混同されていないか
5. ログ向け詳細とユーザー向け文言が適切に分離されているか

「文言が少し気になる」程度ではなく、実際にユーザーや保守担当を誤誘導するかどうかを基準に評価してください。

## 5. テストレビュー方針

テストについては、次を確認してください。

1. 単体テストが入力検証とエラー整形を十分にカバーしているか
2. API 結合テストが正常系と異常系の契約を確認しているか
3. E2E テストが UI 上の重要な失敗ケースを確認しているか
4. 失敗時の表示が、実際の想定文言や構造に対して検証されているか
5. テストが不安定になりやすい構成や、環境依存の強すぎる箇所がないか

## 6. ドキュメントレビュー方針

README と詳細ドキュメントについては、次を確認してください。

1. 現在の実装と設定値に一致しているか
2. セットアップ手順が実際に再現可能か
3. トラブルシューティングが実装上の失敗ケースと整合しているか
4. ユーザー向け説明と UI の挙動が一致しているか

## 7. 出力形式

レビュー結果は、問題点を重要度の高い順に列挙してください。
指摘がある場合は、各項目で次を簡潔に示してください。

1. 問題の概要
2. なぜ問題なのか
3. 影響範囲
4. 修正の方向性
5. 該当箇所

実害やリスクがあるものを優先し、単なる好みの指摘は後回しにしてください。

## 8. 問題が見つからない場合

問題が見つからなかった場合でも、次は必ず示してください。

1. 問題なしと判断した理由
2. 未確認事項
3. 残るリスクやレビューの限界

## 9. 最後のまとめ

最後に、全体の総評を短くまとめてください。
必要なら、優先して直すべき上位 1 から 3 件も示してください。
